## Клиент (LibreOffice)

### «Все серверы недоступны» (сообщение у сотрудника)
Проверить со стороны администратора:
1. Запустить диагностический макрос: **Сервис → Макросы → Мои макросы и диалоги →
   My Macros → ai_macro → Health → AICheckServer → Запустить**. Смотрим HTTP-код
   каждого адреса.
   - `200` → сервер живой, проблема в формате ответа или модели. Смотрите `logs/ai_suggester.local.log`.
   - `000` / пусто → `curl` не смог подключиться. Сервер не запущен, адрес в `Settings.xba`
     неверный или firewall блокирует.
   - `502/503/504` → сервер перегружен; перезапустить или увеличить `NUM_THREADS`.
2. Убедиться на сервере: `systemctl status ai-suggester` (или открытое окно `start.sh`)
   и `curl http://localhost:8000/health`.
3. Если URL надо сменить — править `Settings.xba`, пересобирать .oxt и рассылать
   заново (см. [`ADMIN_GUIDE.md`](ADMIN_GUIDE.md)).

### «Выделенный фрагмент пустой»
Макрос работает только с **выделенным** текстом. Выделите фрагмент в документе,
затем нажмите «AI: Улучшить текст».

### Правки не отображаются как Track Changes
- В модуле `ai_macro.Settings` должно быть `GetUseTrackChanges = True` (по умолчанию).
- В LibreOffice документе: меню **Правка → Отслеживать изменения → Запись** не обязательно включать
  вручную — макрос включит автоматически и восстановит ваше исходное состояние после применения.
- Если правки всё-таки применяются сразу без track changes: у вас LibreOffice <7.0 или
  документ защищён паролем от редактирования. Снимите защиту через **Правка → Отслеживать изменения → Защитить**.

### Ошибка «Доступ к временной папке запрещён»
Макрос пишет в системный `Temp`. Если антивирус блокирует — добавьте `%TEMP%\ai_*.txt`
в исключения. На Astra Linux доступ должен быть всегда.

### Кириллица приходит как `??????`
Проверьте, что `curl` системный, а не ASCII-only (`curl --version` должен содержать `LibIDN`).
В Windows 10 1809+ и любом Linux этого не происходит.

## Серверы (FastAPI)

### `ModuleNotFoundError: No module named 'shared'`
Запускайте из корня репозитория: `uvicorn main:app` выполняйте внутри папки
`local_server/` или `fastapi_server(2)/`. Скрипт `start.sh`/`start.bat` делает это автоматически.
Если запускаете вручную — сначала выполните `cd local_server`.

### `[Errno 98] Address already in use`
На порту 8000 уже висит другой процесс.
```bash
# Linux
ss -tlnp | grep 8000
kill <PID>

# Windows
netstat -ano | findstr :8000
taskkill /PID <PID> /F
```
Или поменять порт в `start.sh`/`.env`.

### Локальный сервер отвечает, но модель долго молчит
Первый запрос после старта Ollama прогревает модель в RAM (≈ 30–60 с для 30B).
Последующие запросы быстрее. Проверить:
```bash
curl http://localhost:11434/api/tags   # модель в списке?
ollama ps                               # модель в памяти?
```

### «`ОШИБКА: Ollama недоступна`»
Сервис Ollama не запущен.
- Windows: запустить приложение Ollama из меню «Пуск».
- Linux: `sudo systemctl start ollama` или `nohup ollama serve &`.

### Модель даёт «неправильный формат» часто
Обычно при `qwen3:30b-a3b`/`qwen2.5:32b` это <5% запросов. Если >20%:
- Понизить температуру: `"temperature": 0.05` в `main.py`.
- Проверить, что промт не обрезан (параметр `num_ctx` увеличить до 8192).
- Убедиться, что модель та же, что рекомендована. Не подходят: модели <7B, модели без русской поддержки.

### Облачный сервер: «`HTTP 401`»
Ключ OpenRouter истёк или недействителен. Сгенерировать новый: <https://openrouter.ai/keys>.
Положить в `fastapi_server(2)/.env`. **Никогда** не коммитить в git.

### Облачный сервер: «Все модели вернули 429»
Rate limit бесплатного тарифа OpenRouter исчерпан. Подождать час или подключить платный.
Альтернатива: переключиться на локальный режим (`SERVER_LIST=http://localhost:8000/suggest` в Settings.xba).

## RAG

### `ollama: model 'nomic-embed-text' not found`
```bash
ollama pull nomic-embed-text
```

### Добавление документа зависло
Эмбеддинг каждого чанка занимает 1–3 с на CPU. Документ в 200 страниц (~200 чанков) — 3–10 минут.
Следите за прогрессом в консоли (после фикса в этом PR `rag_cli add` показывает stat в конце).

### `ValueError: После очистки документ пуст`
Файл состоит целиком из служебки (например, только оглавление, или PDF с вшитыми картинками).
Попробуйте вручную скопировать содержание в `.txt` и загрузить.

### Документы проиндексировались, но модель их не использует
1. Убедиться, что `RAG_ENABLED=true` в `.env`.
2. Перезапустить локальный сервер.
3. В `/metrics` смотреть `rag_documents` — число должно совпасть с `rag_cli list`.
4. В `/suggest` промт будет длиннее — это подтверждает, что RAG-контекст добавился.

### Поиск возвращает нерелевантные результаты
- Увеличить `RAG_TOP_K` до 6–8.
- Проверить через `rag_cli search "фраза"` — если всё равно не находит, вероятно,
  нужен более качественный эмбеддер. Доступные альтернативы:
  - `ollama pull bge-m3` (768-dim, мультиязычный, лучше nomic на русском).
  - `ollama pull mxbai-embed-large` (1024-dim, чуть медленнее).
  Указать новый в `.env`: `RAG_EMBED_MODEL=bge-m3`.

## Логи и аудит

### Логи не пишутся
Проверить `LOG_DIR`. По умолчанию `logs/`, относительно рабочего каталога uvicorn.
Если сервер запускается через systemd, использовать абсолютный путь: `LOG_DIR=/var/log/ai_suggester`.

### БД аудита заблокирована (`database is locked`)
Возникает, если две копии сервера пишут в одну БД. Использовать разные `AUDIT_DB` в двух `.env`.

### Место на диске заканчивается
Проверить `LOG_RETENTION_DAYS`. Стандарт 30 суток — для активной эксплуатации
разумно, но при 1000 запросов/день аудит может вырасти до сотен МБ.
Уменьшите до 7–14 дней или увеличьте дисковое пространство.
