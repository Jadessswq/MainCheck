# Обучение модели на ведомственных документах (RAG)

На конфигурации 32 ГБ RAM / 32 ядра / без GPU **полноценный fine-tuning** LLM
(дообучение всех весов) на нормативных документах технически невозможен —
требуется 80+ ГБ VRAM или распределённая CPU-тренировка на десятки серверов.

Поэтому используется **RAG** (Retrieval-Augmented Generation) — модель не переобучается,
а в момент каждого запроса получает релевантные фрагменты из ваших ведомственных
документов в виде контекста. Это даёт ≈90% эффекта fine-tuning при нулевой
вычислительной стоимости обучения.

---

## 1. Что делает RAG-слой

Папка (по умолчанию `data/docs/`) → для каждого файла:

1. **Извлекает текст.** Поддерживаются `.txt`, `.docx`, `.pdf`, `.htm/.html`, `.rtf`, `.md`.
2. **Очищает служебные элементы Гарант/КонсультантПлюс:**
   - Шапки «Система ГАРАНТ», «КонсультантПлюс надёжная правовая поддержка»,
     «Документ предоставлен КонсультантПлюс».
   - Подписи «© Гарант, 2024», копирайты.
   - «Дата печати: …», «Дата сохранения: …».
   - Номера страниц «Страница N из M», «12 / 48».
   - Блоки «Информация об изменениях», «См. также», «См. комментарии».
   - Оглавления с точками-отточиями (строки вида `1.2.3. Название.......123`).
   - Мягкие переносы посередине слова (`орга-\nнизация` → `организация`).
   - Неразрывные/тонкие пробелы, soft hyphen, BOM, разрывы страниц.
   - Нормализация `NFC` **без** `NFKC`, чтобы не потерять `№` (NFKC превращает его в «No»).
3. **Режет на перекрывающиеся чанки** (~1200 символов, перекрытие 150).
4. **Считает эмбеддинги** локальной моделью Ollama `nomic-embed-text` (≈ 100 МБ, 768 dim,
   хорошо понимает русский).
5. **Сохраняет** в `data/rag_store/` (JSON + JSONL, ничего тяжёлого, не требует ChromaDB/FAISS).

Во время запроса `/suggest` сервер:
- берёт выделенный пользователем текст,
- ищет `RAG_TOP_K` наиболее похожих чанков (косинусная близость),
- добавляет их в промт как блок **«ПРИМЕНИМЫЕ НОРМАТИВНЫЕ ФРАГМЕНТЫ»** — модель
  учитывает их как справку, но не цитирует напрямую в блоке CHANGES.

## 2. Включение

В `Сервер/local/.env`:

```env
RAG_ENABLED=true
RAG_STORE_DIR=data/rag_store
RAG_DOCS_DIR=data/docs
RAG_TOP_K=4
RAG_EMBED_MODEL=nomic-embed-text
```

Предварительно скачать эмбеддер:
```bash
ollama pull nomic-embed-text
```

Перезапустить `Сервер/local` (`start.sh` / `start.bat`).

## 3. Загрузка документов (CLI)

Все команды запускаются из корня репозитория.

### Добавить документ
```bash
PYTHONPATH=Сервер python -m shared.rag_cli add ./path/to/fz_44.docx --doc-id fz-44 --version 2024-07
```

Поля:
- `--doc-id` — уникальный идентификатор документа. Если не задан, берётся имя файла.
- `--version` — строка версии (дата, номер редакции). Если не задана, подставляется текущая метка времени.

Вывод:
```json
{
  "doc_id": "fz-44",
  "path": "./path/to/fz_44.docx",
  "version": "2024-07",
  "chunks": 142,
  "embedder": "ollama:nomic-embed-text",
  "text_len": 185231,
  "stats": {
    "original_lines": 4021,
    "kept_lines": 3120,
    "removed_service": 901,
    "joined_wrapped": 217,
    "reduction_pct": 22.4
  }
}
```

### Заменить документ новой редакцией
Просто вызвать `add` с тем же `--doc-id` и новой `--version`:
```bash
PYTHONPATH=Сервер python -m shared.rag_cli add ./path/to/fz_44_new.docx --doc-id fz-44 --version 2025-03
```
Старые чанки автоматически удаляются перед добавлением новых — дублирования не происходит.

### Посмотреть, что загружено
```bash
PYTHONPATH=Сервер python -m shared.rag_cli list
```

### Удалить документ (например, норма отменена)
```bash
PYTHONPATH=Сервер python -m shared.rag_cli remove fz-44
```

### Проверить качество поиска
```bash
PYTHONPATH=Сервер python -m shared.rag_cli search "согласно распоряжения" --top-k 3
```

### Массовая загрузка из папки
Удобно для первоначального наполнения. Папка сканируется рекурсивно,
`doc_id` берётся из относительного пути:

```bash
PYTHONPATH=Сервер python -m shared.rag_cli ingest-folder ./data/docs
```

## 4. Workflow с Гарант/КонсультантПлюс

1. Из клиента Гарант/КонсультантПлюс сохранить норматив в файл
   (правый клик → «Экспорт в Word / Печать в файл»).
2. Положить файл в `data/docs/` (или любую другую папку).
3. Выполнить:
   ```bash
   PYTHONPATH=Сервер python -m shared.rag_cli add data/docs/fz_44.docx --doc-id fz-44 --version 2025-03
   ```
4. Через минуту (эмбеддинги считаются на CPU, ~1–3 с/чанк на nomic-embed-text) —
   документ доступен модели.

### Что делать, когда норма обновилась
- Обновлённый файл положить в `data/docs/` с **таким же именем и тем же `--doc-id`**.
- Повторно выполнить `add` — старая редакция автоматически удаляется перед индексацией новой.
- Альтернатива: явно удалить старую через `remove`, затем `add` новую.

### Что делать, когда норма отменена
```bash
PYTHONPATH=Сервер python -m shared.rag_cli remove fz-44
```

## 5. Офлайн / резервный режим (без Ollama)

Если на рабочей станции нет Ollama, RAG может работать с детерминированным
хеш-эмбеддером (TF-like, без нейросети). Качество ниже, чем у `nomic-embed-text`,
но поиск по лексическому пересечению работает. Используется в тестах и как fallback.

```bash
PYTHONPATH=Сервер python -m shared.rag_cli --embedder hashing add file.txt --doc-id test
```

## 6. Запросы и ответы

После включения RAG клиентский код в LibreOffice менять не нужно — сервер
сам добавляет контекст в промт. Видно это будет:
- в логе `logs/ai_suggester.local.log` — вырастет длина сообщения,
- в результатах — модель начнёт ссылаться на нормы (косвенно, в пояснениях).

### Сброс всего индекса
```bash
rm -rf data/rag_store
```
или просто переместить папку — `RAG_STORE_DIR` создаётся заново при первом `add`.

## 7. Метрики RAG

`GET /metrics` на локальном сервере возвращает:

```json
{
  "server": "local",
  "model": "qwen2.5:32b",
  "rag_enabled": true,
  "rag_documents": 37,
  "audit": { ... }
}
```

Для анализа качества поиска используйте `python -m shared.rag_cli search "…"` —
видно, какие документы попадают в топ-k, и можно оценить релевантность.
